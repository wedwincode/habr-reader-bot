# Habr reader bot

Легковесный сервис для Docker, который:

1. периодически подтягивает новые закладки из Habr в markdown файл,
2. каждый день присылает в Telegram верхнюю непрочитанную статью,
3. по кнопке `Я прочитал` отмечает запись в markdown как прочитанную.

## Как это работает

Файл должен содержать задачи в таком формате:

```md
- [ ] [Почему я перестал держать БД в голове и сделал Structly](https://habr.com/ru/articles/1011576/)
- [x] [Мечтают ли папки с заметками об отдельном бэкенде? От ресерча к MVP](https://habr.com/ru/articles/1012030/)
```

Новые закладки добавляются в самый верх.

## Что нужно подготовить

- создать Telegram-бота через BotFather;
- узнать свой `chat_id`;
- смонтировать в контейнер локальный путь с git-репозиторием, где лежит markdown файл;
- указать `HABR_SOURCE_URL`.

## Важный момент по Habr

Предпочтительный вариант - использовать RSS ссылку из раздела закладок Habr, если она у тебя доступна. В профиле Habr на страницах есть действие `Copy RSS link`, значит у соответствующих разделов RSS предусмотрен интерфейсом сайта.

Если RSS не подходит, можно использовать обычную HTML-страницу закладок. Для приватных закладок нужно передать cookie в `HABR_COOKIE_HEADER`.

## Запуск

```bash
cp .env.example .env
# отредактируй .env

docker compose up -d --build
```

## Команды бота

- `/start` - справка
- `/next` - показать следующую непрочитанную статью
- `/sync` - вручную синхронизировать закладки
- `/done <url>` - отметить статью прочитанной

## Git

По умолчанию бот просто редактирует файл на диске.

Если включить:

```env
GIT_AUTO_COMMIT=true
GIT_REPO_PATH=/data/notes
```

то после изменений бот сделает commit и попробует `push` в указанный remote/branch.

## Ограничения

- html-структура Habr может меняться, поэтому RSS стабильнее;
- если ты вручную меняешь файл, бот корректно возьмет следующую непрочитанную запись при следующем уведомлении;
- сейчас это один процесс с polling, без внешней базы, чтобы было легче крутить на Orange Pi.
